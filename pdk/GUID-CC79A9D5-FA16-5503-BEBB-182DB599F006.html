Concepts

This section describes the technology, architecture and behavior of the Domain Manager.

  • Architecture
    This topic describes the architecture of the domain manager.
  • Domain
    A domain is associated with one or more applications, typically the applications that take part in, or are associated with those activities.
  • Domain Management Policy DLL
    This provides an implementation of the domain tree, and a suitable implementation of the DmPolicy class that the domain manager uses to access that tree.
  • Power State Transitions
    This topic describes power state transitions and how domains behave when these happen.
  • Dynamic Behavior
    This topic describes the dynamic behavior of the domain manager for state transitions.
