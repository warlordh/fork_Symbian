Composite File System

Provides a single point of access for read-only flash based devices.

Introduction

The composite file system unifies the file systems for flash devices (ROM, ROFS and/or FAT file systems) into a common file system that is used with drive Z:. The file server will direct requests to the required file system as required.

Background information

The code required by a device is supplied on some form of ROM or Flash memory. The composite file system combines the file systems that could be used to store the required code into one universal filing system which is used to access drive Z:.

Composite features

The following are features of the composite file system:

  • Combines the ROM, ROFS and FAT filing systems into one universal filing system for drive Z.

  • The file system is designed to be read-only.
