Receives next message. No memory is allocated during the operation of this method.

Parameters

TRequestStatus & aStatusThis request will be completed once message has been received.
TInt & aMessageSizeTells the size of the received message. If bigger than the size of the aMessagePayload, then only the beginning of the received message is copied to aMessagePayload. This means that either Receive must be called with bigger buffer to receive full message or Skip must be called to indicate that the full message is not needed.
TDes8 & aMessagePayloadStorage to the message to be received. The format of the message is determined by the category Uid used in Open method.

Skip()

voidSkip()[pure virtual]

Skip one (i.e. next) message. This method should be called only if Receive completion indicates that there is not enough space in aMessagePayload. Of course, you should attempt to recover, i.e. try to call Receive with big enough aMessagePayload buffer before calling this method.
