User to User Signalling Service Settings Tutorial

This tutorial describes how to get and set the User to User Signalling (UUS) service settings.

User-To-User Signalling (UUS) is a supplementary service that allows a subscriber to send and receive a limited amount of subscriber generated User to User Information (UUI) to and from another user in an association with a call to the user. Three variants on the UUS service, UUS1-3, are available.

The UUS settings enable a user to accept or reject all incoming UUS requests. These are phone-level settings and the user cannot modify them on a per-call basis.

  • Get the device UUS settings using RMobilePhone::GetUUSSetting(). UUS settings are described by RMobilePhone::TMobilePhoneUUSSetting.
  • Set the device UUS settings using RMobilePhone::SetUUSSetting()

  • Use RMobilePhone::NotifyUUSSettingChange() to get the notification of any changes in the UUS settings.
